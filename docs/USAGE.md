# git-gui 使用指南

git-gui 是一个 Rust 实现的 git 客户端，主打"可见性"+"安全网"，对标 JetBrains git 体验。

## 快速开始

### 启动 TUI

```bash
cd ~/work/git-gui/crates/tui
cargo run
```

或编译后运行：

```bash
cargo build --release
cd 你的git仓库
~/work/git-gui/target/release/tui
```

## 主界面：Status 面板

启动后默认进入 Status 面板，显示：
- 当前分支
- upstream 分支
- 领先/落后提交数
- 工作区状态（干净/有改动）
- 冲突文件列表（如果有）

### 可用键位

| 键位 | 功能 | 说明 |
|------|------|------|
| `j` / `k` / `↑` / `↓` | 滚动 | 状态文本过长时滚动查看 |
| `s` | Stage | 进入文件暂存视图 |
| `b` | Branch | 分支列表（创建/切换/删除） |
| `h` | Stash | stash 列表（创建/应用/弹出/丢弃） |
| `S` | Submodule | 子仓库列表 |
| `l` | Log | 查看提交历史 |
| `d` | Diff | 查看改动 / 逐 hunk 暂存 |
| `p` | Push | 推送（非快进自动整合后重推） |
| `u` | Update | 拉取远端（自动 stash，进度条显示，可 `Esc` 取消） |
| `m` | 策略 | 切换 merge / rebase 整合策略 |
| `R` | Resume | 恢复未完成的冲突解决 |
| `r` | Refresh | 刷新状态 |
| `q` | Quit | 退出程序 |

> 所有视图支持**鼠标滚轮 / 触摸板**上下滚动：看 diff 的内容区（diff 右栏 / log 详情右栏）纯滚动、不移光标；文件树 / 列表则移动选中。启用鼠标后若需终端原生选择/复制文本，按住 `Shift` 再拖选。

## Stage 视图：文件暂存与提交

按 `s` 从 Status 进入 Stage 视图。文件以**可折叠目录树**呈现：同目录的文件归并到目录节点下，可对单个文件、也可对整个目录批量操作。

### 文件状态标识

- `●` 绿色：已暂存
- 红色：已修改（未暂存）
- `?` 灰色：未跟踪
- `◐` 黄色：部分暂存（既有暂存又有未暂存改动）
- `▾` / `▸` 青色：目录（已展开 / 已折叠）

### 可用键位

| 键位 | 功能 | 说明 |
|------|------|------|
| `j` / `k` / `↑` / `↓` | 上下移动 | 移动光标到上/下一行 |
| `l` / `→` | 展开目录 | 展开光标所在目录 |
| `h` / `←` | 折叠目录 | 折叠光标所在目录 |
| `Enter` | 展开/折叠 | 切换光标所在目录 |
| `Space` | 切换暂存 | 文件：暂存/取消暂存；目录：对其下全部文件批量暂存（全部已暂存时则批量取消） |
| `a` | 全暂存 | 暂存所有改动文件 |
| `d` | 回滚 | 丢弃文件/整目录的改动（含未跟踪文件），二次确认后执行 |
| `c` | 提交 | 进入提交消息输入 |
| `q` / `Esc` | 返回 | 返回 Status 面板 |

### 提交流程

1. 用 `j/k` 移动到要暂存的文件或目录
2. 按 `Space` 暂存（文件或整目录；或 `a` 全暂存）
3. 按 `c` 进入提交消息输入
4. 输入消息，按 `Enter` 完成提交
5. 按 `Esc` 取消提交

### 回滚改动（安全网）

把光标停在文件或目录上按 `d`，二次确认（`y` 确认 / 其他键取消）后回滚其改动。回滚**不是直接删除**：改动会先 `stash` 兜底一份（含未跟踪文件），万一误操作可到 Stash 视图 `pop` 找回。

## Log 视图：查看提交历史

按 `l` 从 Status 进入 Log 视图。左侧蓝色图形列由 `git log --graph` 生成，展示分支/合并拓扑；`j`/`k` 在提交间移动会自动跳过纯图形连接行。

### 列表模式

默认显示最近 50 条提交，**滚动到接近底部时自动加载更多**（每次 +50，重拉保证分支图整体一致）。每行 = 左侧分支图形列（蓝色）+ 提交信息。每条提交包括：
- SHA（黄色，前 8 位）
- 提交消息（第一行）
- 作者 + 相对时间（灰色）

### 可用键位

| 键位 | 功能 | 说明 |
|------|------|------|
| `j` / `k` / `↑` / `↓` | 上下移动 | 移动光标/滚动 |
| `Enter` | 查看详情 | 进入双栏详情（左侧提交信息+文件目录树，右侧内容/diff） |
| `y` | 复制哈希 | 复制当前提交的完整 SHA 到系统剪贴板 |
| `q` / `Esc` | 返回 | 返回 Status 面板 |

### 详情模式（双栏）

按 `Enter` 进入。左栏是「提交信息」+ 改动文件的**目录树**，右栏显示当前项内容（提交信息全文，或该文件的 diff，着色）。

- 左栏（目录树）：`j`/`k` 切项，`l`/`Enter` 进右栏（目录则展开）、`h` 折叠目录，`q` 返回列表
- 右栏（内容）：`j`/`k` 滚动，`h`/`q` 回左栏
- `y`：复制该提交的完整 SHA 到剪贴板（两栏下均可）

## Diff 视图：左侧文件目录树 + 右侧单文件 diff

按 `d` 从 Status 进入。左栏把改动文件按**目录树**展示（可折叠），右栏显示当前文件的 hunk diff（着色：`+` 绿 / `-` 红 / `@@` 块头品红），可逐 hunk 或逐行暂存/取消暂存。聚焦栏边框为青色。

### 可用键位

| 焦点 | 键位 | 功能 |
|------|------|------|
| 文件树 | `j` / `k` | 移动（目录 / 文件） |
| 文件树 | `l` / `Enter` | 文件 → 进右栏 diff；目录 → 展开 |
| 文件树 | `h` | 目录 → 折叠 |
| 文件树 | `Space` | 暂存/取消整个文件 或 整个目录 |
| 文件树 | `t` | 切换未暂存 / 已暂存视图 |
| 文件树 | `q` / `Esc` | 返回 Status |
| diff | `j` / `k` | 移动光标 |
| diff | `Space` | `+`/`-` 行：选中该行；`@@` 块头：暂存整 hunk |
| diff | `s` | 暂存（未暂存视图）/ 取消暂存（已暂存视图）选中的行 |
| diff | `h` / `q` | 回到文件树 |

- 未暂存视图 `Space`/`s` = 暂存；已暂存视图（`t` 切换后）= 取消暂存
- 逐行选择限定在单个 hunk 内（换 hunk 选行会清空上一处选择）
- 未跟踪文件无 hunk 概念，仍在 Stage 视图整文件暂存

## Branch 视图：分支管理

按 `b` 从 Status 进入，列出本地分支（当前分支高亮）：

| 键位 | 功能 |
|------|------|
| `j` / `k` / `↑` / `↓` | 上下移动 |
| `Enter` | 切换到选中分支 |
| `c` | 创建新分支（输入名字 → Enter） |
| `d` | 删除选中分支（确认后） |
| `q` / `Esc` | 返回 Status |

## Stash 视图：储藏管理

按 `h` 从 Status 进入，列出所有 stash：

| 键位 | 功能 |
|------|------|
| `j` / `k` / `↑` / `↓` | 上下移动 |
| `c` | 创建 stash（输入消息，含 untracked） |
| `a` | apply（应用，保留 stash） |
| `P` | pop（弹出 = apply + drop） |
| `d` | drop（丢弃，确认后） |
| `q` / `Esc` | 返回 Status |

## Update：安全拉取

按 `u` 执行 Update（pull），自动处理脏工作区：

1. **自动 stash**：有未提交改动时自动 `stash --include-untracked`
2. **fetch + 整合**：按当前策略 merge / rebase（带 `ignore-space-change`），按 `m` 切换
3. **自动还原**：整合成功后自动 `stash apply` + `drop`
4. **冲突处理**：整合冲突 → 自动进入三栏冲突视图

Update 在后台线程执行，界面不冻结；fetch 阶段可按 `Esc` 取消（取消发生在 stash 之前，工作区不受影响），状态栏显示进度。

### Push 结果

按 `p` 推送（后台执行，状态栏显示进度，可 `Esc` 取消）：

- **推送成功**：显示"推送成功"
- **无 upstream**：提示先执行 `git push -u origin <branch>`
- **非快进（远端领先）**：自动整合后重推 —— 无冲突全自动；有冲突进解决视图，解决后自动重推；取消/失败则不推、不丢改动

## 冲突解决视图

Update 遇到冲突时自动进入三栏视图。

### 布局

```
文件概览条: 文件1(✓) | 文件2(!) | 文件3(?)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Ours    │    Base    │   Theirs
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- 绿色文本：行级魔法棒自动定夺的行
- 黄色文本：需要人工选择的冲突行

### 可用键位

| 键位 | 功能 | 说明 |
|------|------|------|
| `j` | 下一块 | 移动到下一个冲突块 |
| `k` | 上一块 | 移动到上一个冲突块 |
| `o` | 选 Ours | 保留本地改动 |
| `t` | 选 Theirs | 保留远端改动 |
| `b` | 选 Base | 恢复共同祖先 |
| `←` / `→` | 切换选择 | 在 Ours/Base/Theirs 之间移动选择 |
| `J` / `K` | 块内滚动 | 当前冲突块内纵向滚动 |
| `n` | 下一文件 | 切换到下一个冲突文件 |
| `p` | 上一文件 | 切换到上一个冲突文件 |
| `w` | 写回 | 保存当前文件的选择并 `git add` |
| `c` | 完成 | 所有冲突解决后，继续整合 |
| `x` | 放弃 | 放弃整合，回退到 Update 前 |
| `q` / `Esc` | 退出 | 退出程序（保留冲突状态） |

### 行级魔法棒

git-gui 实现了行级冲突自动定夺：

- **单边改动**：只有一边改了某行 → 自动采纳该改动（绿色标注）
- **真冲突**：两边都改了同一行 → 保留冲突让用户选择（黄色标注）

例如：
```
<<<<<<< ours
A           ← 只有 ours 改了这行 → 自动保留
B           ← 两边都改了 → 需要人工选择
4
||||||| base
2
3
4
=======
2
C           ← 两边都改了 → 需要人工选择
D           ← 只有 theirs 改了这行 → 自动保留
>>>>>>> theirs
```

结果：只剩中间一行需要人工选择，其余自动定夺。

## 崩溃恢复

如果在冲突解决过程中退出程序（按 `q` 或意外崩溃），下次启动时：

1. Status 面板会显示"冲突文件: N 个(按 R 恢复解决)"
2. 按 `R` 自动恢复到冲突视图
3. 之前的选择状态和 autostash 都会保留

## rerere：冲突重放

git-gui 支持 git rerere（记住冲突解决方案）：

- 第一次解决冲突 → 自动记录
- 下次遇到相同冲突 → 自动应用之前的解法

## CLI 模式

除了 TUI，git-gui 也提供 CLI 模式：

```bash
# 查看状态
cargo run -- status

# 执行 update
cargo run -- update
```

## 完整工作流示例

### 场景 1：日常提交推送

```
1. 启动 TUI
2. 按 s → Stage 视图
3. 按 Space 暂存改动的文件（或 a 全暂存）
4. 按 c → 输入 commit 消息 → Enter 提交
5. 按 q/Esc → 返回 Status
6. 按 p → 推送
```

### 场景 2：拉取远端有冲突

```
1. 按 u → Update
2. 自动进入冲突视图
3. 用 j/k 浏览冲突块
4. 用 o/t/b 选择保留哪边
5. 按 w 保存当前文件
6. 按 n 切换到下一个文件（如果有）
7. 按 c 完成整合
```

### 场景 3：查看提交历史

```
1. 按 l → Log 视图
2. 用 j/k 浏览提交列表
3. 按 Enter 查看某个提交的详情
4. 按 q/Esc 返回列表
5. 按 q/Esc 返回 Status
```

## 特性总结

- ✅ **零外部依赖**：gitcore 只依赖 std + spawn git
- ✅ **安全 pull**：自动 stash/unstash，不丢改动
- ✅ **行级魔法棒**：冲突块拆分到行级，自动定夺单边改动
- ✅ **多文件导航**：冲突文件间自由切换，独立保留进度
- ✅ **崩溃恢复**：检测未完成的整合并恢复
- ✅ **rerere 集成**：自动重放冲突解法
- ✅ **完整测试**：44 个测试覆盖核心流程

## 架构

```
git-gui/
├── crates/gitcore/     # 核心库（零依赖）
│   ├── update.rs       # Update 状态机
│   ├── resolve.rs      # 冲突解析 + 行级魔法棒
│   ├── stage.rs        # 暂存文件
│   ├── commit.rs       # 提交
│   ├── push.rs         # 推送
│   ├── log.rs          # 提交历史
│   └── diff.rs         # diff 查看
└── crates/tui/         # ratatui 前端
    ├── app.rs          # 主事件循环 + 状态面板
    ├── conflict_ui.rs  # 三栏冲突视图
    ├── stage_ui.rs     # Stage 视图
    ├── log_ui.rs       # Log 视图
    ├── branch_ui.rs    # Branch 视图
    ├── stash_ui.rs     # Stash 视图
    ├── submodule_ui.rs # Submodule 视图
    ├── keys.rs         # 方向键→控制字符归一化
    └── scroll.rs       # 跟随光标自动滚动
```
