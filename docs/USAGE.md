# git-gui 使用指南

git-gui 是一个 Rust 实现的 git 客户端，主打"可见性"+"安全网"，对标 JetBrains git 体验。

## 快速开始

### 启动 TUI

```bash
cd ~/work/git-gui/crates/tui
cargo run
```

或编译后运行：

```bash
cargo build --release
cd 你的git仓库
~/work/git-gui/target/release/tui
```

## 主界面：Status 面板

启动后默认进入 Status 面板，显示：
- 当前分支
- upstream 分支
- 领先/落后提交数
- 工作区状态（干净/有改动）
- 冲突文件列表（如果有）

### 可用键位

| 键位 | 功能 | 说明 |
|------|------|------|
| `s` | Stage | 进入文件暂存视图 |
| `l` | Log | 查看提交历史 |
| `d` | Diff | 查看工作区改动 |
| `p` | Push | 推送到远端 |
| `u` | Update | 拉取远端（自动 stash） |
| `R` | Resume | 恢复未完成的冲突解决 |
| `r` | Refresh | 刷新状态 |
| `q` | Quit | 退出程序 |

## Stage 视图：文件暂存与提交

按 `s` 从 Status 进入 Stage 视图。

### 文件状态标识

- `●` 绿色：已暂存
- 红色：已修改（未暂存）
- `?` 灰色：未跟踪
- `◐` 黄色：部分暂存（既有暂存又有未暂存改动）

### 可用键位

| 键位 | 功能 | 说明 |
|------|------|------|
| `j` | 下移 | 移动光标到下一个文件 |
| `k` | 上移 | 移动光标到上一个文件 |
| `Space` | 切换暂存 | 暂存/取消暂存当前文件 |
| `a` | 全暂存 | 暂存所有改动文件 |
| `c` | 提交 | 进入提交消息输入 |
| `q` | 返回 | 返回 Status 面板 |

### 提交流程

1. 用 `j/k` 移动到要暂存的文件
2. 按 `Space` 暂存（或 `a` 全暂存）
3. 按 `c` 进入提交消息输入
4. 输入消息，按 `Enter` 完成提交
5. 按 `Esc` 取消提交

## Log 视图：查看提交历史

按 `l` 从 Status 进入 Log 视图。

### 列表模式

显示最近 50 条提交，每条包括：
- SHA（黄色，前 8 位）
- 提交消息（第一行）
- 作者 + 相对时间（灰色）

### 可用键位

| 键位 | 功能 | 说明 |
|------|------|------|
| `j` | 下移 | 移动到下一条提交 |
| `k` | 上移 | 移动到上一条提交 |
| `Enter` | 查看详情 | 显示完整提交内容（message + diff） |
| `q` | 返回 | 返回 Status 面板 |

### 详情模式

按 `Enter` 查看提交详情，显示 `git show` 的完整输出。

- 按 `q` 返回列表

## Diff 视图：查看改动

按 `d` 从 Status 查看工作区改动（工作区 vs 暂存区）。

- 按 `q` 返回 Status 面板

## Update：安全拉取

按 `u` 执行 Update（pull），自动处理脏工作区：

1. **自动 stash**：有未提交改动时自动 `stash --include-untracked`
2. **fetch + 整合**：默认 merge（带 `ignore-space-change`）
3. **自动还原**：整合成功后自动 `stash apply` + `drop`
4. **冲突处理**：整合冲突 → 自动进入三栏冲突视图

### Push 结果

按 `p` 推送时可能遇到：

- **推送成功**：显示"推送成功"
- **无 upstream**：提示先执行 `git push -u origin <branch>`
- **非快进**：提示先 pull

## 冲突解决视图

Update 遇到冲突时自动进入三栏视图。

### 布局

```
文件概览条: 文件1(✓) | 文件2(!) | 文件3(?)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
   Ours    │    Base    │   Theirs
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

- 绿色文本：行级魔法棒自动定夺的行
- 黄色文本：需要人工选择的冲突行

### 可用键位

| 键位 | 功能 | 说明 |
|------|------|------|
| `j` | 下一块 | 移动到下一个冲突块 |
| `k` | 上一块 | 移动到上一个冲突块 |
| `h` | 选 Ours | 保留本地改动 |
| `l` | 选 Theirs | 保留远端改动 |
| `b` | 选 Base | 恢复共同祖先 |
| `n` | 下一文件 | 切换到下一个冲突文件 |
| `p` | 上一文件 | 切换到上一个冲突文件 |
| `w` | 写入 | 保存当前文件的选择并 `git add` |
| `Enter` | 继续 | 完成所有冲突解决，继续整合 |
| `a` | 放弃 | 放弃整合，回退到 Update 前 |
| `q` | 退出 | 退出程序（保留冲突状态） |

### 行级魔法棒

git-gui 实现了行级冲突自动定夺：

- **单边改动**：只有一边改了某行 → 自动采纳该改动（绿色标注）
- **真冲突**：两边都改了同一行 → 保留冲突让用户选择（黄色标注）

例如：
```
<<<<<<< ours
A           ← 只有 ours 改了这行 → 自动保留
B           ← 两边都改了 → 需要人工选择
4
||||||| base
2
3
4
=======
2
C           ← 两边都改了 → 需要人工选择
D           ← 只有 theirs 改了这行 → 自动保留
>>>>>>> theirs
```

结果：只剩中间一行需要人工选择，其余自动定夺。

## 崩溃恢复

如果在冲突解决过程中退出程序（按 `q` 或意外崩溃），下次启动时：

1. Status 面板会显示"冲突文件: N 个(按 R 恢复解决)"
2. 按 `R` 自动恢复到冲突视图
3. 之前的选择状态和 autostash 都会保留

## rerere：冲突重放

git-gui 支持 git rerere（记住冲突解决方案）：

- 第一次解决冲突 → 自动记录
- 下次遇到相同冲突 → 自动应用之前的解法

## CLI 模式

除了 TUI，git-gui 也提供 CLI 模式：

```bash
# 查看状态
cargo run -- status

# 执行 update
cargo run -- update
```

## 完整工作流示例

### 场景 1：日常提交推送

```
1. 启动 TUI
2. 按 s → Stage 视图
3. 按 Space 暂存改动的文件（或 a 全暂存）
4. 按 c → 输入 commit 消息 → Enter 提交
5. 按 q → 返回 Status
6. 按 p → 推送
```

### 场景 2：拉取远端有冲突

```
1. 按 u → Update
2. 自动进入冲突视图
3. 用 j/k 浏览冲突块
4. 用 h/l/b 选择保留哪边
5. 按 w 保存当前文件
6. 按 n 切换到下一个文件（如果有）
7. 按 Enter 完成整合
```

### 场景 3：查看提交历史

```
1. 按 l → Log 视图
2. 用 j/k 浏览提交列表
3. 按 Enter 查看某个提交的详情
4. 按 q 返回列表
5. 按 q 返回 Status
```

## 特性总结

- ✅ **零外部依赖**：gitcore 只依赖 std + spawn git
- ✅ **安全 pull**：自动 stash/unstash，不丢改动
- ✅ **行级魔法棒**：冲突块拆分到行级，自动定夺单边改动
- ✅ **多文件导航**：冲突文件间自由切换，独立保留进度
- ✅ **崩溃恢复**：检测未完成的整合并恢复
- ✅ **rerere 集成**：自动重放冲突解法
- ✅ **完整测试**：33 个测试覆盖核心流程

## 架构

```
git-gui/
├── crates/gitcore/     # 核心库（零依赖）
│   ├── update.rs       # Update 状态机
│   ├── resolve.rs      # 冲突解析 + 行级魔法棒
│   ├── stage.rs        # 暂存文件
│   ├── commit.rs       # 提交
│   ├── push.rs         # 推送
│   ├── log.rs          # 提交历史
│   └── diff.rs         # diff 查看
└── crates/tui/         # ratatui 前端
    ├── app.rs          # 主事件循环
    ├── conflict_ui.rs  # 三栏冲突视图
    ├── stage_ui.rs     # Stage 视图
    └── log_ui.rs       # Log 视图
```
